关于tigervnc_server启动时kde突然出现欢迎界面问题

tiandic Lv1
  2026-01-25 13:48:54 2026-01-25 13:48:54 Created   2026-01-25 16:42:14 2026-01-25 16:42:14 Updated  

前言

为了问题可复现
我使用了

  1. VMware-workstation-full-17.6.3 来自https://github.com/201853910/VMwareWorkstation/releases/download/17.0/VMware-workstation-full-17.6.3-24583834.x86_64.bundle
  2. debian-13 来自BT种子https://tiandic.github.io/project/raw/debian-13.3.0-amd64-DVD-1.iso.torrent

接下来所有操作都将进行逐步说明,如果不说明,则是选择默认的选项

安装debian-13虚拟机

  • 选择使用ISO映像,填入iso文件的路径
  • 在选择客户机系统中选择debian 12.x 64-bit
  • 在debian安装菜单中选择图形化安装
  • 在Set up users and passwords中输入用户名a,密码同样是a
  • 在Software selection中取消勾选GNOME,勾选KDE Plasma
  • 在Install the GRUB boot loader中选择/dev/sda(虽然这有点废话,但这里不能狂按回车选默认,所以就说一下)

安装vnc server与ssh server

  • 在/etc/apt/sources.list中注释唯一的一行,并填入网络源:

安装指定版本的tigervnc

  • sudo apt update && sudo apt install openssh-server tigervnc-standalone-server=1.15.0+dfsg-2 -y

启动vnc服务

在桌面的中,启动konsole,输入vncserver命令启动vnc服务

随后突然出现启动Debian输入密码登陆成功后出现的欢迎界面

然后恢复正常

问题排查

使用
sudo journalctl -f
然后再次执行vncserver

观察到日志出现:

Jan 25 00:29:04 debian systemd[1079]: Reload requested from client PID 2155 (‘x-session-manag’) (unit app-org.kde.konsole@202af4a920744d6b8cccf4d3c1086b05.service)…

猜测是由PID为2155的进程触发的该问题

使用ps -ef --forest查看调用链与x-session-manager的启动参数

观察到调用链:

可以看到是我们执行的vncserver触发了x-session-manager的调用

并且x-session-manager没有被传递任何命令行参数,那么应该就是通过环境变量传递的了

尝试复现

问题复现成功!

所以可以确定,就是我们调用vncserver时,vncserver调用了x-session-manager,从而启动的欢迎界面

那么x-session-manager是什么?
通过ls命令可以看到:

x-session-manager实际上是/usr/bin/startplasma-x11

那么究竟是哪个环境变量触发的呢?

通过简单的组合搭配可以发现最小复现命令

为了排除前面tigervnc-server的影响,让我们重启Debian

然后使用 Xephyr 创建一个虚拟 X server尝试复现问题

问题再次复现成功!

所以可以确定,是WAYLAND_DISPLAY=wayland-0

因为KDE默认为wayland,所以会设置WAYLAND_DISPLAY=wayland-0

所以可能是XWayland的一个BUG

如果登陆时更换X11登陆则不会出现该问题
