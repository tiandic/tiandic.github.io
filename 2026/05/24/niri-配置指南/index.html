前言

niri的配置文件使用 KDL 编写,路径为~/.config/niri/config.kdl
其核心概念为上下工作区与卷轴式平铺窗口,一个工作区内竖对齐的窗口统称为列(如果竖直方向只有一个窗口,也可以称为列)

kdl快速入门

由于该语言不像yaml,json等一样知名度极高,所以在此简单介绍,虽然很简单,但用来做niri的配置足够了.

概览

以下是常用的顶层配置选项

  • binds {}: 快捷键设置
  • layout {}: 窗口样式
  • workspace "<name>" {}: 命名工作区
  • spawn-at-startup与spawn-sh-at-startup: 随着niri启动而一起启动
  • cursor {}: 光标样式
  • window-rule {}: 设置指定程序窗口的配置

窗口规则

这里不仅能配置指定窗口,而且一些全局样式配置也在这里

在window-rule {}中,你首先要使用match选择一些窗口,当然,你也可以不选择,如果不选择,那么该配置将全局生效

也可以使用多个匹配项,窗口必须与所有匹配项都匹配,该配置才能生效

可以通过运行 niri msg pick-window 并单击相关窗口来查找窗口的标题和应用程序ID

可以看到,从KDL语法来讲,每一个匹配项都是match节点的属性
以下是常用的匹配项:

  • match title="Firefox": 按照窗口标题匹配 (匹配是动态的,所以当窗口标题变化后,会重新开始匹配规则)
  • match app-id="Alacritty": 按照窗口ID匹配
  • match is-active=true: 是否为活动窗口 (每隔工作区都有一个活动窗口,所以他不是唯一的)
  • match is-focused=false: 是否为焦点窗口 (这是唯一的,即使有多个工作区,焦点窗口也只有一个)
  • match is-floating=true: 是否为浮动窗口
  • match is-window-cast-target=true: 是否为当前正在进行的窗口屏幕录制的窗口
  • match is-urgent=true: 是否为紧急窗口

以下是常用的配置项:

窗口打开

  • default-column-width { proportion 0.75; }: 设置窗口的默认宽
  • default-window-height { fixed 500; }: 设置窗口的默认高
  • open-on-workspace "chat": 在命名工作区 “chat” 打开
  • open-maximized-to-edges true: 最大化打开 (相当于点一下窗口标题栏的最大化按钮)
  • open-maximized true: 列模式最大化打开 (这个选项与上面选项的区别在于,他不会将窗口完全填满显示器)
  • open-fullscreen true: 全屏打开 (与open-maximized-to-edges的区别在于,他不会显示窗口的标题栏)
  • open-floating true: 以浮动窗口打开
  • open-focused false: 窗口在打开时是否自动获得焦点

动态属性

  • draw-border-with-background false: 是否绘制活动窗口的背景 (例如: 打开kitty后,你也许会看到原本黑色的背景在窗口获得焦点后,覆盖上了一层蓝色,设置该属性为false可以让kitty不被覆盖背景)
  • opacity 0.5: 不透明度, 0.0 表示完全透明, 1.0 表示完全不透明. (此设置会叠加在窗口自身的透明度之上,因此半透明窗口会变得更加透明)
  • block-out-from "screencast": 不要让这个窗口出现在”屏幕共享/录屏”中 (注意: 并未对第三方截图工具进行屏蔽,如果在录屏过程中打开带有预览功能的截图工具,则被屏蔽的窗口仍会在录屏视频中显示)
  • block-out-from "screen-capture": 阻止窗口被任何”屏幕捕获”方式获取 (比上面的范围更大)
  • default-floating-position x=100 y=200: 窗口在浮动布局下打开或移动到浮动布局时的初始位置, 之后,窗口会记住它上次的浮动位置
  • geometry-corner-radius 12: 对焦环/边框 圆角半径 (也可以单独设置每个角geometry-corner-radius 8 8 0 0, 顺序与 CSS 相同: 左上角,右上角,右下角,左下角)
  • clip-to-geometry true: 将内容裁剪到窗口几何区域(geometry)内,其常与geometry-corner-radius配合使用,但设置了geometry-corner-radius但没设置该选项为true时,窗口内的的直角会覆盖 对焦环/边框 的部分区域,导致不美观 (可以试试打开firefox或者dolphin,保持geometry-corner-radius 12,更改clip-to-geometry的值,对比一下前后的窗口角落)
  • baba-is-float true: 让你的窗户可以上下浮动 (这是2025年愚人节的特别功能,不实用,但很好玩 :D 演示视频)
  • 对焦环:
  • 边框(他与对焦环的区别在于,边框会在没有焦点的窗口周围绘制一圈边框,而对焦环不会):

命名工作区

命名工作区可以让我们将一些窗口分类在不同的工作区,例如: 将聊天应用与代码工作区分开在不同工作区,当我们希望给朋友发信息时,只需要切换工作区即可,而无需频繁按下Alt+tab,以寻找聊天窗口

所以他通常与window-rule配合使用,以实现指定程序打开时,在命名工作区打开,而非在当前工作区打开

且命名工作区也可以单独使用子节点layout设置样式,layout的选项与顶级 layout {} 相同

另外: Mod 在TTY 终端上运行 niri 时等同于 Win ,在嵌套的 winit 窗口中运行 niri 时等同于 Alt 所以,在配置niri桌面时,将其认为是Win 即可

常用的修饰符

  • Ctrl 或 Control
  • Shift
  • Alt
  • Super 或 Win
  • Mod

推荐使用Mod替代Super 与 Win ,因为这更具兼容性

常用的action

截图
  • screenshot : 打开内置的交互式截图界面
  • screenshot-screen 与 screenshot-window : 分别截取当前聚焦屏幕或窗口的屏幕截图
窗口关闭与状态
  • close-window: 关闭当前窗口
  • fullscreen-window: 全屏 (相当于播放视频时,点击’全屏’的效果,让窗口画面铺满整个屏幕,没有标题栏)
  • toggle-windowed-fullscreen: 窗口化全屏 (让应用以为你是在全屏显示它, 实际上还是一个窗口, 其效果大多数时候相当于”去除当前窗口的标题栏”)
  • toggle-window-floating: 在浮动/平铺窗口之间切换 (如果你不知道什么是浮动窗口的话,不妨在一个窗口中使用Win+V ,然后使用Alt+Tab不断切换到其他窗口试试看)
  • move-window-to-floating: 移动到浮动层
  • move-window-to-tiling: 移动到平铺层
聚焦(Focus)
窗口 (作用范围仅限当前工作区)
  • focus-window: 按 id 聚焦窗口
  • focus-window-up: 切换到上面的窗口 (注意: 这无法切换到上面的工作区的窗口)
  • focus-window-down: 切换到下面的窗口 (注意: 这无法切换到下面的工作区的窗口)
  • focus-window-previous: 切换到上一个聚焦的窗口
  • focus-window-top: 切换到最上窗口
  • focus-window-bottom: 切换到最下窗口
列 (作用范围仅限当前工作区)
  • focus-column-left: 切换到左列 (其实就是切换到左边窗口 :D)
  • focus-column-right: 切换到右列 (其实就是切换到右边窗口 :D)
  • focus-column-first/last: 首列/末列
  • focus-column: 按索引聚焦列
工作区
  • focus-workspace: 按索引/名称切换工作区 (常与命名工作区结合使用)
  • focus-workspace-up: 切换到上面的工作区
  • focus-workspace-down: 切换到下面工作区
  • focus-workspace-previous: 上一个聚焦的工作区
移动窗口 / 列 / 工作区
窗口移动
  • move-window-up: 聚焦窗口在列中上移
  • move-window-down: 聚焦窗口在列中下移
  • move-window-to-workspace <name or number>: 聚焦窗口移动到指定工作区
列移动
  • move-column-left: 窗口左移
  • move-column-right: 窗口右移
  • move-column-to-workspace <name or number>: 当前工作区的整个列到指定工作区
工作区移动
  • move-workspace-up: 工作区上移
  • move-workspace-down: 工作区下移
列布局控制
  • set-column-width "<number>": 设置列宽
  • maximize-column: 最大化列
  • center-column: 居中列
窗口尺寸
  • set-window-width "<number>": 设置窗口宽
  • reset-window-height: 重置窗口高
  • switch-preset-window-width: 切换到预设的窗口宽
  • switch-preset-window-height: 切换到预设的窗口高

样式

这里是关于窗口样式与配置的选项

常用的节点有:

  • gaps 16: 窗口间的距离
  • center-focused-column "never": 何时在切换焦点时使列居中 其值有"never": 不进行特殊居中, always: 焦点列将始终居中显示
  • always-center-single-column: 始终居中单列 (优先级比center-focused-column更高)
  • empty-workspace-above-first: 在工作区的最上方添加空工作区
  • default-column-width { proportion 0.33333; }: 默认列宽, proportion为在屏幕大小中的占比
