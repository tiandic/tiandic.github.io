前言

grub主题很多,但关于主题制作的中文资料寥寥无几,于是本文就诞生了

grub的主题文件由两部分构成

  • 全局属性
  • 组件

而决定主题整体的美观程度的只有两个:

  • 背景图 (全局属性)
  • 启动菜单 (组件)

我们将从这两个配置开始,逐步引入其他全局属性与组件, 最终构建出如下可用主题:

启动菜单的位置与大小

接下来,让我们配置启动菜单

启动菜单的可配置项较多,让我们从其位置与大小开始配置

我们知道,配置一个矩形,一般需要设置四个值

  • 矩形左上角的x坐标
  • 矩形左上角的y坐标
  • 矩形的宽
  • 矩形的高

配置启动菜单也类似,但略有不同:

  • left: 菜单左边界相对于屏幕左边界的距离 (实际配置效果与x坐标相同)
  • top: 菜单上边界相对于屏幕上边界的距离 (实际配置效果与y坐标相同)
  • width: 菜单的宽
  • height: 菜单的高
    其值均可使用相对于屏幕的百分比来配置

启动菜单的样式框

接下来,让我们了解并添加启动菜单的样式框,以区分出启动菜单的位置与范围

样式框由9个矩形区域组成

每个区域对应一张极小的图像,grub会缩放他们

例如: 与West (w)区域对应的图像boot_menu_w.png, 这个区域是启动菜单的左侧的边

我们知道,矩形的边是一条线,这条线的任意部分的图像都是相同的,而boot_menu_w.png就是这条线的一小部分,grub会不断重复这一小部分,拼接这些小部分,直到长度与启动菜单的边相同,然后渲染这条边

或者,更准确地描述缩放规则
缩放规则如下：

  1. 边缘切片（北、南、东、西）会沿其相邻边缘的方向进行缩放。例如，西侧切片会沿垂直方向缩放。
  2. 角片（西北、东北、东南和西南）未进行缩放。
  3. 中间的切片按比例缩放，以填充中间剩余的空间。

那么,让我们从 boot_menu.7z 下载提前制作好的9个区域的图像 (稍后会说明如何制作)

制作方法:
Inkscape 是一款可缩放矢量图形编辑器，是创建样式化方框图像的非常实用的工具。将图形切片成所需图像切片的一种有效方法是：

  1. 创建或打开您想要使用的绘图。
  2. 在图层堆栈顶部创建一个新图层，并将其设置为可见。将该图层选为当前图层。
  3. 在图形上绘制 9 个矩形，作为切片的位置。清除填充选项，并将描边设置为 1 像素宽的实线描边。切片的角必须精确对齐；如果偏差一个像素，在 GRUB 菜单中渲染样式框时可能会很明显。您可能需要转到“文件”|“文档属性”|“网格”，启用网格或创建参考线（单击图形旁边的标尺并拖动到图形上；松开鼠标按钮以放置参考线），以帮助您精确放置矩形。
  4. 右键单击中心切片矩形，选择“对象属性”。将“Id”更改为“slice_c”，然后单击“设置”。对其余 8 个矩形重复此操作，根据其位置分别赋予它们“slice_n”、“slice_ne”、“slice_e”等 Id 值。
  5. 保存图纸。
  6. 选中所有切片矩形。选中切片图层后，只需按 Ctrl+A 即可选中所有矩形。状态栏应显示已选中 9 个矩形。
  7. 在图层面板中，单击切片图层的图层隐藏图标。即使矩形被隐藏，它们仍将保持选中状态。
  8. 选择“文件”|“导出位图”，勾选“批量导出 9 个选定对象”复选框。确保“隐藏除选定对象外的所有对象”复选框未勾选。单击“导出”。这将在与绘图相同的目录中创建 PNG 文件，文件名称以切片名称命名。现在，这些文件可用于 GRUB 主题中的样式框。

所以grub主题配置文件的编写其实不难,难的是如何制作一个好看的背景图与美观的样式框

接下来的内容将会简单介绍其他组件与全局属性,这些内容很少会有主题用到,所以仅作了解即可

所有全局属性

  • title-font: 标题字体, 其值要求与item_font相同
  • desktop-image: 背景图
  • desktop-image-scale-method: 指定 desktop-image 的缩放方式，可选值为 stretch、crop、padding、fitwidth、fitheight。stretch 为拉伸填满屏幕；crop 将图像局部等比缩放以裁剪填满屏幕；padding 将完整图像等比缩放并完整显示在屏幕内；fitwidth 等比缩放使图像宽度与屏幕宽度对齐；fitheight 等比缩放使图像高度与屏幕高度对齐。默认值为 stretch。
  • desktop-image-h-align: 背景图水平对齐方式,如果 desktop-image-scale-method 不等于stretch，则指定 desktop-image 的水平对齐方式。选项包括left、center和right。默认值为center。
  • desktop-image-v-align: 如果 desktop-image-scale-method 不等于“stretch”，则指定 desktop-image 的垂直对齐方式。选项包括top、center和bottom。默认值为center。
  • terminal-border: 终端窗口的边框宽度
  • terminal-left: 终端窗口的x坐标
  • terminal-top: 终端窗口的y坐标
  • terminal-width: 终端窗口的宽度
  • terminal-height: 终端窗口的高度

所有组件

以下是所有组件及其支持属性的列表。

label（标签）

标签用于显示一行文本。

属性说明
id设置为 "__timeout__" 时，显示距默认条目自动启动的已用时间。
text要显示的文本。若 id 设为 "__timeout__" 且未设置 text，则显示剩余秒数。若设为 "@KEYMAP_SHORT@"、"@KEYMAP_MIDDLE@" 或 "@KEYMAP_LONG@"，则显示预定义的热键信息。
font用于文本显示的字体。
color文本颜色。
align文本在组件内的水平对齐方式，可选值为 "left"（左对齐）、"center"（居中）和 "right"（右对齐）。
visible设为 "false" 时隐藏标签。

image（图像）

用于显示图像的组件，图像会被缩放以适应组件大小。

属性说明
file要加载的图像文件的完整路径。

progress_bar（进度条）

显示一个水平方向的进度条。可使用简单的纯色填充矩形渲染，也可使用一对像素图样式的盒子渲染。

属性说明
id设置为 "__timeout__" 时，显示距默认条目自动启动的已用时间。
fg_color纯色渲染模式下的前景色。
bg_color纯色渲染模式下的背景色。
border_color纯色渲染模式下的边框颜色。
text_color文本颜色。
bar_style进度条边框的样式盒子规范，例如 "progress_frame_*.png"。若值等于 highlight_style，则不显示样式盒子。
highlight_style进度条高亮区域的样式盒子规范，例如 "progress_hl_*.png"。该盒子仅用于绘制高亮区域，随进度完成而增大。若值等于 bar_style，则不显示样式盒子。
highlight_overlay设为 "true" 时，高亮盒子的侧边切片将覆盖在边框盒子的侧边切片上，以实现圆角进度条效果。默认值为 "false"。
font进度条使用的字体。
text在进度条上显示的文本。若进度条 ID 设为 "__timeout__"，且此属性值为 "@TIMEOUT_NOTIFICATION_SHORT@"、"@TIMEOUT_NOTIFICATION_MIDDLE@" 或 "@TIMEOUT_NOTIFICATION_LONG@"，则 GRUB 将随倒计时更新此属性为提示信息。

circular_progress（圆形进度指示器）

显示一个圆形进度指示器。该组件的外观由两张图像决定：中心图像（center）和刻度图像（tick）。中心图像通常较大，绘制在组件中央；刻度图像则根据组件属性，沿组件内圆周绘制若干次。

属性说明
id设置为 "__timeout__" 时，显示距默认条目自动启动的已用时间。
center_bitmap绘制在组件中央的图像文件名。
tick_bitmap刻度标记的图像文件名。
num_ticks构成完整圆圈的刻度总数。
ticks_disappear布尔值，表示随 value 趋近 end，刻度是逐渐出现还是逐渐消失。可设为 "true" 或 "false"，默认为 "false"。
start_angle首个刻度出现或消失的位置。以”鹦鹉”（parrot）为单位，1 parrot = 完整圆圈的 1/256。也可使用 "xxx deg" 或 "xxx °" 设置角度。

boot_menu（启动菜单）

显示 GRUB 启动菜单，允许用户选择并执行启动项。

属性说明
item_font菜单项标题使用的字体。
selected_item_font选中菜单项的字体，设为 "inherit"（默认）则与 item_font 相同。
item_color菜单项标题的颜色。
selected_item_color选中菜单项的颜色，设为 "inherit"（默认）则与 item_color 相同。
icon_width菜单项图标的宽度，图标会缩放至指定大小。
icon_height菜单项图标的高度。
item_height每个菜单项的高度（像素）。
item_padding菜单项内容四周的留白（像素）。
item_icon_space菜单项图标与标题文字之间的间距（像素）。
item_spacing菜单项之间的间距（像素）。
menu_pixmap_style菜单边框样式盒子的图像文件匹配模式，例如 "menu_*.png"。
item_pixmap_style菜单项样式盒子的图像文件匹配模式。
selected_item_pixmap_style选中项高亮样式盒子的图像文件匹配模式。
scrollbar布尔值，表示是否在已配置边框和滑块样式盒子的情况下绘制滚动条。
scrollbar_frame整个滚动条的图像文件匹配模式，例如 "scrollbar_*.png"。
scrollbar_thumb滚动条滑块（随滚动移动的部分）的图像文件匹配模式，例如 "scrollbar_thumb_*.png"。
scrollbar_thumb_overlay设为 "true" 时，滑块侧边切片将覆盖在边框侧边切片上，实现圆角滚动条效果。默认为 "false"。
scrollbar_slice绘制滚动条的菜单边框切片区域，可选值为 "west"、"center"、"east"（默认）。详见下方说明。
scrollbar_left_pad滚动条左侧内边距（像素），scrollbar_slice 为 "west" 时无效。
scrollbar_right_pad滚动条右侧内边距（像素），scrollbar_slice 为 "east" 时无效。
scrollbar_top_pad滚动条顶部内边距（像素）。
scrollbar_bottom_pad滚动条底部内边距（像素）。
visible设为 "false" 时隐藏启动菜单。

scrollbar_slice 说明：

  • "west"：滚动条绘制在西侧切片（右对齐）
  • "east"：滚动条绘制在东侧切片（左对齐）
  • "center"：滚动条绘制在中央切片。若显示滚动条，菜单项宽度将缩减滚动条宽度；若不显示，菜单项宽度即为中央切片宽度。使用中央切片时无需菜单像素图盒子。

canvas（画布）

画布是一种允许在其中手动放置子组件的容器。它不会改变子组件的位置，并为每个子组件分配其首选大小。

hbox（水平布局盒）

hbox 容器从左到右依次排列子组件，每个子组件使用其首选宽度。每个子组件的高度设置为所有子组件首选高度的最大值。

vbox（垂直布局盒）

vbox 容器从上到下依次排列子组件，每个子组件使用其首选高度。每个子组件的宽度设置为所有子组件首选宽度的最大值。

通用属性

以下属性被所有组件支持：

属性格式说明
leftx、p%、p%+x容器左边框到对象左边框的距离。可为像素值、百分比，或两者的混合。
topx、p%、p%+x容器上边框到对象上边框的距离，格式同上。
widthx、p%、p%+x对象的宽度，格式同上。
heightx、p%、p%+x对象的高度，格式同上。
id任意字符串组件的标识符，可由脚本用于引用 GUI 组件树中的各个组件。

特殊 ID 值：

ID说明
"__timeout__"带有此 ID 的组件将由 GRUB 更新，用于显示距默认条目自动启动的已用时间。适用组件：label、circular_progress、progress_bar。
